# Prompt para o Codex adaptar o site do curso Método Strong

Você vai atuar como um engenheiro front-end sênior e copy-focused.

Sua missão é **adaptar e melhorar o site atual do curso Método Strong**, mantendo a base existente, sem quebrar a estrutura, o JavaScript, a navegação, os anchors, os botões e o fluxo de conversão.

## Contexto
O projeto atual já possui uma landing page funcional em `index.html` com seções como:
- hero
- transformação
- módulos
- provas
- oferta
- FAQ
- CTA final

O site é de um curso de edição de vídeos chamado **Método Strong**, com foco em:
- iniciantes
- criadores de conteúdo
- freelancers iniciantes
- pessoas que querem aprender edição com clareza, ritmo, legenda, narrativa e acabamento

A promessa principal é:
- sair do zero
- aprender a editar com critério
- parar de depender de tutorial solto
- transformar edição em habilidade útil e até vendável

## Objetivo da adaptação
Quero que você **melhore o site atual**, deixando ele:
- mais profissional
- mais vendedor
- mais claro
- mais organizado por módulos
- com melhor hierarquia visual
- com copy mais forte
- com melhor leitura no mobile
- sem perder performance

## Regra mais importante
**Não recrie o projeto do zero.**
Você deve **adaptar o arquivo existente**, preservando ao máximo:
- estrutura HTML atual
- classes e IDs existentes sempre que possível
- comportamento do JS
- links, CTAs e âncoras
- organização geral da landing page

## Restrições técnicas
- Não remover seções importantes sem substituição melhor.
- Não quebrar responsividade.
- Não alterar anchors já usados no menu, como `#transformacao`, `#modulos`, `#provas`, `#oferta`, `#faq`.
- Não remover CTA de WhatsApp.
- Não remover modal de vídeo se ele já estiver funcional.
- Não trocar a proposta do produto.
- Não inventar bibliotecas novas sem necessidade.
- Não deixar o site mais pesado sem motivo.
- Não depender de build tool se o projeto atual não usa isso.

## O que precisa ser melhorado

### 1. Hero section
Melhore o hero para ficar mais forte comercialmente.

Quero:
- headline mais impactante
- subheadline mais clara
- mais percepção de valor logo acima da dobra
- CTA principal muito claro
- CTA secundário coerente
- benefícios mais objetivos
- menos sensação de texto genérico

A headline deve vender:
- clareza
- ritmo
- edição com critério
- aprendizado para quem está começando

Evite headline vaga.

### 2. Seção de módulos
A seção de módulos precisa parecer mais “curso de verdade” e menos apenas “cards bonitos”.

Quero que você reorganize os módulos com lógica didática e percepção de progressão.

Base sugerida:
- Módulo 1 — Fundamentos para sair do zero
- Módulo 2 — Ritmo, retenção e narrativa
- Módulo 3 — Legenda, leitura e destaque
- Módulo 4 — Acabamento visual e valor percebido
- Módulo 5 — Adaptação para diferentes formatos
- Módulo 6 — Edição como habilidade aplicável e vendável

Cada módulo deve comunicar:
- o que o aluno aprende
- por que isso importa
- qual transformação ele gera

### 3. Copy geral
Reescreva a copy do site com mais força.

Direção:
- menos enrolação
- menos frase genérica
- mais clareza de benefício
- mais linguagem de transformação
- mais autoridade prática
- mais conexão com dores reais de quem quer editar melhor

Evitar:
- promessas exageradas
- frases vazias
- texto inflado
- blocos longos demais

### 4. Oferta
A seção de oferta precisa ficar mais convincente.

Melhore:
- percepção do que o aluno recebe
- empilhamento de valor
- clareza do preço
- argumentos para compra agora
- bônus mais visíveis
- sensação de oportunidade

### 5. FAQ
Ajuste o FAQ para responder melhor objeções reais, como:
- sou iniciante
- preciso de PC forte?
- serve para reels e vídeos longos?
- dá para usar para freela?
- o curso é prático?
- posso estudar no meu ritmo?

### 6. Autoridade e prova
Se a seção de provas já existir, reforce melhor o posicionamento.

Quero que a autoridade seja apresentada de forma mais estratégica:
- experiência prática
- projetos reais
- exigência de resultado
- aprendizado aplicado no mercado real

Sem transformar isso em promessa enganosa.

### 7. UX/UI
Melhore a experiência visual mantendo o projeto leve.

Diretrizes:
- melhorar hierarquia entre títulos, textos e cards
- melhorar espaçamentos
- reduzir sensação de poluição
- melhorar leitura no mobile
- melhorar escaneabilidade
- dar mais contraste para CTAs
- deixar a página com cara mais premium

### 8. Mobile first
O mobile precisa ser prioridade.

Quero atenção especial para:
- conteúdo principal acima da dobra
- CTA aparecer cedo
- títulos sem quebrar feio
- cards com boa leitura
- espaçamento consistente
- botões fáceis de tocar

## Entregáveis
Quero que você me devolva:

### A. Código adaptado
- versão final do `index.html` ajustada
- alterações mínimas e inteligentes em CSS se necessário
- ajustes em JS apenas se realmente necessário

### B. Resumo das alterações
Explique objetivamente:
- o que foi alterado
- por que melhora conversão
- qualquer risco técnico
- pontos que ficaram preservados

## Critérios de qualidade
Sua resposta precisa seguir isto:
- preservar compatibilidade com a base atual
- melhorar conversão
- melhorar clareza da oferta
- melhorar leitura no mobile
- manter consistência visual
- manter funcionamento da página
- não quebrar classes, IDs ou scripts sem necessidade

## Instrução final
Faça a adaptação com mentalidade de **otimização de landing page**, não de redesign artístico gratuito.

O foco é:
- vender melhor
- comunicar melhor
- parecer mais profissional
- manter o projeto funcional

Se precisar trocar textos, troque.
Se precisar reorganizar blocos, reorganize.
Mas faça isso com o mínimo de risco técnico possível.
